Purpose

This Module Manager Demo and the related module provide a comprehensive  example of ThreadX Module Manager feature. The Module data area isolation is also shown exploiting the MPU.

The demo is built exploiting the STMCube FW and its HAL, BSP and drivers stack. Also the Microsoft ThreadX RTOS and some of related middleware are included.


STMCube is an STMicroelectronics original initiative to ease developers life by reducing development efforts, time and cost.

STM32Cube covers STM32 portfolio.



The drivers provided within this package support the  STM32U585.

Update History

Main Changes

  • The current demo, with respect to the original  demo available in STM32Cube_FW_U5_V1.1.0 (STM32Cube_FW_U5_V1.1.0\Projects\NUCLEO-U575ZI-Q\Applications\ThreadX\Tx_MPU), was extended adding:
  • The FileX middleware with FTP client, DHCP client and SNTP client. 
  • The LevelX middleware
  • The  NOR Flash and the related QSPI and DMA drivers 

Contents

Projects

The application is able to execute the module "in place" from internal flash or from SRAM and "from file" downloading it from the FTP server and saving it on the file system. 

The module debugging capability is also shown (limited to module "execution in place" from internal flash and to IAR EWARM debugger)

This demo use HAL drivers, LL drivers, BSP drivers from STMCube and Azure RTOS from MS. The demo is provided only with IAR EWARM project.



  • Notes:
  • This demo do not use Trust Zone, anyway for more details about TZ please refer to AN5347

Known Limitations

  • BSP MXCHIP: Access point mode with the TCP/IP mode on STM32 host is not functional. It works when TCP/IP runs on the EMW3080B module
  • The NUCLEO-U575ZI-Q\Applications\ThreadX\Tx_MPU Project Architecture with STM32CubeIDE is not aligned with recommended STM32CubeIDE Architecture.(Will be reworked for next Release V1.2.0)
